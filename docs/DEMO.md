## Reproduce the recording-tool outputs

From `/Users/fabiencampana/Documents/ship-ready`:

```bash
pnpm install
pnpm shipready gui
```

In a second shell:

```bash
pnpm demo:fodmapp:captions
pnpm demo:fodmapp:record
pnpm demo:fodmapp:compose
```

The default GUI URL is `http://127.0.0.1:4317`; override it for recording with `SHIPREADY_GUI_URL`. `compose` checks for `/opt/homebrew/bin/ffmpeg`. Outputs go to `validation/demo-fodmapp-recording-v2/`.

Optional synthetic voice generation:

```bash
ELEVENLABS_API_KEY=... ELEVENLABS_VOICE_ID=... pnpm demo:fodmapp:voice
pnpm demo:fodmapp:compose
```

Do not commit keys or place them in demo artifacts. If either variable is absent, voice generation skips. The package scripts reproduce tooling outputs; human approval/provenance for the final voice package remains documented in `validation/demo-fodmapp-voiceover-final/`.

## Recording-only behavior

- Browser captions are demo tooling injected by `scripts/demo/recordFodmappDemo.ts`; they are not a GUI feature.
- Deterministic clipboard success is recording-only. The recording context grants clipboard permission and stubs `navigator.clipboard.writeText`.
- Voiceover is optional media generated by `scripts/demo/generateFodmappVoiceover.ts`; it is not product behavior.
- Recording and composition write media under `validation/`; they do not write the inspected repository.
- The GUI remains preview/copy-only and calls only `POST /api/ui-report`.
